# t-report

Прототип системы еженедельной отчётности на основе Excel-источника.

Проект автоматически проходит полный пайплайн обработки данных: читает исходный Excel-файл, проверяет структуру обязательных листов, нормализует строки, выполняет аналитическую обработку записей и собирает итоговый текстовый отчёт в preview-режиме.

Текущая версия ориентирована на локальный запуск и демонстрацию рабочего сквозного контура: **от входного `.xlsx` до готового консольного отчёта**.

---

## Назначение проекта

Система предназначена для автоматизированной сборки еженедельного отчёта по двум наборам данных из Excel:

- записи по КП;
- записи по поставкам.

Проект не сводится к одному большому скрипту: обработка разбита на отдельные этапы, чтобы код было проще развивать, проверять и поддерживать.

---

## Что делает система

Текущая реализация выполняет следующие шаги:

1. читает Excel-файл;
2. ищет обязательные листы;
3. валидирует структуру заголовков;
4. нормализует строки и значения ячеек;
5. выделяет записи по заданным правилам;
6. сортирует результаты;
7. формирует итоговый preview-отчёт;
8. выводит отчёт в консоль.

---

## Архитектурная идея

Проект построен как последовательный пайплайн обработки:

**источник данных → проверка структуры → нормализация → анализ → сортировка → сборка отчёта → вывод**

Такой подход позволяет:
- отделять ответственность модулей;
- локализовать ошибки по этапам;
- дорабатывать систему постепенно;
- использовать один и тот же каркас для дальнейшего расширения.

## Структура проекта

- `report_tool.py` — основная точка входа
- `preview_orchestrator.py` — координация preview-сценария
- `cli_parser.py` — разбор аргументов командной строки
- `report_context_factory.py` — подготовка даты отчёта и временного контекста
- `exel_sourse_reader.py` — чтение Excel-файла и обязательных листов
- `structure_validator.py` — проверка структуры и заголовков
- `kp_normalizer.py` — нормализация строк по КП
- `delivery_normalizer.py` — нормализация строк по поставкам
- `entity_mapper.py` — преобразование нормализованных записей в рабочие сущности
- `kp_analyzer.py` — анализ записей по КП
- `delivery_analyzer.py` — анализ записей по поставкам
- `result_sorter.py` — сортировка итоговых результатов
- `report_builder.py` — сборка итогового текстового отчёта
- `console_output.py` — вывод результата в консоль
